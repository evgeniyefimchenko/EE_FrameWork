Контент

В этом разделе описаны методы создания контента.

  • Пользовательский алгоитм действий
    1. Завести типы свойств: Свойства->Типы->Добавить; При создании добавить поле/поля из которых состоит тип свойства
    2. Добавить свойства: Свойства->Список->Добавить; И названия полей для свойства
    3. Добавить набор свойств: Свойства->Наборы свойств->Добавить; Выбрать свойства который войдут в набор
    4. Добавить тип категории: Категории->Типы->Добавить
    5. Определить категории для материалов: Категории->Список->Добавить; После добавления появятся вкладки Сущности категории и Наборы свойств
    6. Создать сущности для наполнения проекта: Сущности->Список->Добавить
  • Описание элементов и назначение.
    1. Типы свойств - имеют возможность собрать несколько полей в своём содержании. Основная задача это облегчить создание составных свойств. К примеру для сущности(карточки/страницы) Жильё Может быть создан тип Для комнат с несколькими полями (Размер, вид, мебель ...) И присвоен свойству Комнаты
    2. Свойства - представляет элемент или группу элементов Типов свойств для назначения Набору свойств категориям и далее наследованием от категорий сущностями к ним принадлежащими.
