Контент

В этом разделе описаны методы создания и настройки контента с учётом того, что в системе уже реализовано наследование типов категорий.

  • Пользовательский алгоритм действий
    1. Завести типы свойств: Свойства -> Типы -> Добавить.

      При создании укажите название «Типа свойств» и добавьте одно или несколько полей (например, text, hidden, checkbox и др.). Эти поля определяют структуру будущих свойств.

    2. Добавить свойства: Свойства -> Список -> Добавить.

      Каждое свойство основывается на уже созданном «Типе свойств» и использует набор его полей. У свойства указывается видимость: «для страниц», «для категорий» или «для всех».

    3. Добавить набор свойств: Свойства -> Наборы свойств -> Добавить.

      Набор свойств объединяет несколько свойств в логическую группу (например, «Основной контент», «SEO-настройки», «Параметры товаров» и т.д.). В дальнейшем их можно массово назначать типам категорий. К примеру, набор «Контент страницы» может включать свойства «Заголовок», «Краткое описание» и «Дата публикации».

    4. Добавить тип категории: Категории -> Типы -> Добавить.

      Задайте название, описание и при необходимости назначьте «родительский» тип (для наследования) и/или один или несколько «Наборов свойств». Например:

      • Родительский тип: «Общий тип категорий»
      • Собственные наборы свойств: «SEO-настройки», «Основной контент» и т.д.
      В результате новый «дочерний» тип категории унаследует все наборы свойств своего «родителя» и при этом может добавить собственные.

    5. Определить категории для материалов: Категории -> Список -> Добавить.

      После создания категории назначьте ей соответствующий тип. Категория автоматически получит все свойства из назначенных наборов «своего» типа и всех его «родительских» типов (если таковые есть). При этом свойства с видимостью «только для страниц» не будут видны у категории, а свойства «только для категорий» не попадут на страницы (сущности).

    6. Создать сущности для наполнения проекта: Сущности -> Список -> Добавить.

      Страницы, товары и прочие материалы (сущности) наследуют от своей категории только те свойства, что помечены «для страниц» или «для всех». Это упрощает работу и исключает ненужные поля.

  • Описание элементов и назначение.
    1. Типы свойств —

      Это «конструкторы» полей: вы определяете, какие поля (текстовые, чекбоксы, селекты и т.д.) входят в состав типа. К примеру, в «Типе свойств» «SEO» могут быть поля «meta_title», «meta_description», «open_graph_image» и т.п.

    2. Свойства —

      На базе «Типов свойств» создаются конкретные «Свойства», которым присваивается видимость (для страниц, категорий или всех). Например, «SEO для категорий» будет использовать поля из «Типа свойств» SEO, но с видом «только для категорий».

    3. Наборы свойств —

      Это логические группы, объединяющие несколько «Свойств». Один набор может включать свойства из разных «Типов». Например, «Набор для продуктов» может содержать свойства «Основной контент» (с полем «content»), «SEO» (с полями «meta_title», «meta_description») и «Параметры товара» (цену, скидку, SKU и т.д.).

    4. Типы категорий —

      Позволяют кластеризовать категории по разным сегментам (например, «Новости», «Блог», «Товары» и т.д.). Возможно наследование типов, когда «дочерний» тип получает наборы свойств от «родителя» и может добавлять собственные. К примеру:

      • «Тип категорий» «Общая рубрика» (базовые наборы свойств «SEO», «Социальные элементы»)
      • «Тип категорий» «Новости» наследует от «Общей рубрики» и добавляет свой набор «Дата публикации и автор», «Комментарии» и т.д.
      Если категория выбрала тип, у которого есть «родитель», она автоматически унаследует все свойства от родителя и от тех наборов, которые напрямую назначены этому (дочернему) типу.

    5. Категории —

      Создавая категорию и назначая ей тип (включая наследуемый), вы тем самым получаете полный список свойств, который формируется суммарно из «родительского» и «дочернего» типов. Однако свойства, помеченные как «только для страниц», в категории не появляются.

    6. Сущности (страницы, карточки, материалы) —

      Сущности относятся к определённой категории (или категориям). Эти сущности получают «наследство» свойств, которые имеют видимость «для страниц» или «для всех». Если же свойство помечено «только для категорий», то на уровне сущности оно недоступно.

Наследование типов категорий: когда это удобно

  • Сокращение дублирования

    Если несколько типов категорий (например, «Новости», «Блог», «Статьи») имеют похожие наборы свойств, вы можете вынести общие свойства в «родительский» тип и настроить отдельные поля для каждого «дочернего» типа.

  • Упрощение поддержки

    При изменении «родительского» типа (например, добавлении нового SEO-поля) все «дочерние» типы автоматически наследуют его. Это экономит время и снижает риск пропустить какие-то правки.

  • Контроль специфики

    «Дочерний» тип может обладать дополнительными наборами свойств или переопределять свойства (если логика приложения это позволяет). Так вы получите чёткую базу и гибкость при расширении.

Итог: Наличие наследования типов категорий делает систему ещё более гибкой. Вы можете быстро создавать «дочерние» типы, которые берут большую часть свойств у «родителя» и добавляют / отключают лишь нужные наборы.
